Identifying, explaining, and refactoring 10 Code Smells.

1. Switch Statements (OO Abuser)

BEFORE
public double calculateDiscount(String type) {
    switch (type) {
        case "student": return price * 0.90;
        case "senior": return price * 0.85;
        // ... modifications here violate O/C principle
    }
}
AFTER
// Strategy Pattern
public interface DiscountStrategy {
    double applyDiscount(double basePrice);
}

// StudentDiscount.java
public class StudentDiscount implements DiscountStrategy {
    public double applyDiscount(double price) {
        return price * 0.90;
    }
}

2. Feature Envy (Coupler)

BEFORE
// In OrderService.java
public double calculateValue(Book book) {
    // Service obsesses over Book's data!
    return (book.getBasePrice() * book.getMultiplier()) 
           - book.getDamagePenalty();
}
AFTER
// In Book.java (Data owners perform their own logic)
public double calculateCurrentValue() {
    return (this.basePrice * this.multiplier) 
           - this.damagePenalty;
}

// In OrderService.java
currentTotal += book.calculateCurrentValue();
